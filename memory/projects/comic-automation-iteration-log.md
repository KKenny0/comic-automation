## 2026-02-24｜Iteration 1：解说漫方向重构启动

- 背景：
  - 项目从“通用漫剧自动化”转向“解说漫自动化”主航道。
  - 业务重点从可生成，升级到可规模化产能与平台运营适配。

- 目标：
  - 新建独立分支进行架构与流程改造。
  - 先完成设计方案（Design Doc），再进入执行实现。

- 变更：
  - 新分支：`feat/narrated-comic-automation-v1`
  - 新文档：`projects/comic-automation/docs/DD-006-narrated-comic-pivot-design-v1.md`

- 关键决策：
  1. 采用“先音频后画面”作为解说漫默认生产路径。
  2. 保留 timeline 作为事实源，但升级到 v2 字段以适配解说漫。
  3. 引入“钩子密度”作为核心评估指标。

- 风险与回滚：
  - 风险：新流程增加了 TTS/SRT 依赖，可能带来稳定性问题。
  - 回滚策略：保留 v1 兼容路径，必要时可降级为无音频先行模式。

- 结果与指标：
  - 设计阶段已完成；待进入执行阶段验证端到端耗时与成功率。

- 下一步：
  1. 补充 timeline.v2 schema
  2. 增加 step-1.5-audio 工作流
  3. 跑通首条 60-90 秒样片
